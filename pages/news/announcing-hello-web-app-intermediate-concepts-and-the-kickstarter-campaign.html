There were a lot of concepts that didn't make it into the first book due to their complexity, things that most web apps would probably need at some point, like: * Taking payments from customers. * Adding an API. * Allowing user-uploaded images and editing/resizing those images automatically. * Adding alerts and admin emails. There also wasn't a lot of theory in Hello Web App by design, but there are quite a few things that are great to know as you grow as a web app developer, such as: * Database schema design and reducing queries. * Class-based views. * How and why we should read source code. **All of this and more will be in [Hello Web App: Intermediate Concepts](https://hellowebapp.com/intermediate-concepts)**, the perfect follow-up book to those who've gone through the original Hello Web App tutorial or anyone who has built their first web app and are looking for the next step.
